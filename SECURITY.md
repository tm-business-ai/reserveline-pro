# SECURITY

ReserveLine Pro をGitHubへ公開するとき、または本番利用するときの注意点です。

## 秘密情報の扱い

- `.env` はGitHubに公開しないでください。
- LINEアクセストークン、チャネルシークレット、管理者パスワードはコードに直書きしないでください。
- `.env.example` は設定例だけを書くファイルです。実際の秘密情報は入れないでください。
- `data/*.db` などのローカルDBファイルは公開しないでください。

## 管理者パスワード

本番利用時は、`.env` の `ADMIN_PASSWORD` を必ず変更してください。

```env
ADMIN_PASSWORD=change-me
```

上のような初期値のまま公開・運用しないでください。

## デモモードと本番モード

`DEMO_MODE=true` の場合、LINE APIキーがなくてもWebhook風のJSON送信や管理画面から動作確認できます。ポートフォリオ確認や提案用のデモに向いています。

`DEMO_MODE=false` の場合、LINE Messaging APIとの接続を想定します。この場合は `.env` に `LINE_CHANNEL_ACCESS_TOKEN` と `LINE_CHANNEL_SECRET` を設定してください。

## 本番公開時の追加対策

本番公開する場合は、少なくとも次を確認してください。

- HTTPSで公開する
- LINE署名検証を有効にする
- 管理画面へのアクセス制限を行う
- 推測されにくい `ADMIN_PASSWORD` を設定する
- サーバやPaaSの環境変数で秘密情報を管理する
- ローカルDBやログに個人情報を残しすぎない

このリポジトリはポートフォリオ・MVP用途を想定しています。本番運用では、認証、権限管理、監査ログ、バックアップなどを追加してください。
