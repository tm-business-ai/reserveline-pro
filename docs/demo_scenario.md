# 動作確認シナリオ

ReserveLine Pro の画面と予約受付の動きを確認するための手順です。

## 事前準備

`.env.example` を `.env` にコピーし、`ADMIN_PASSWORD` を設定します。`.env` はGitHubに公開しないファイルです。

```powershell
copy .env.example .env
```

日付例は実行日によって過去日になる場合があります。予約登録を試すときは、必ず現在日以降で、営業時間内かつ営業日の日時を指定してください。

## 1. 管理画面を起動する

```powershell
streamlit run ui/admin_app.py
```

## 2. ログインする

画面に「管理画面ログイン」が表示されたら、`.env` の `ADMIN_PASSWORD` を入力します。

## スクリーンショット撮影ポイント

GitHub公開用に管理画面のスクリーンショットを撮影する場合は、次の画面を確認します。実在の顧客名、電話番号、メールアドレス、LINEユーザーID、APIキー、パスワードは写さず、サンプル太郎、demo-user-001 などの確認用データを使ってください。

- 管理画面トップ
- 本日の予約
- 今週の予約
- 電話・店頭予約の登録
- 予約データをダウンロード
- 予約メニューの設定
- 予約前のお知らせ予定
- LINE予約の動作確認

## 3. 確認用の予約データを作成する

サイドバーの「確認用の予約データを作成」を押します。

次のメニューを使った予約例が作成されます。

- 30分相談
- 60分相談
- 初回カウンセリング

既に同じ確認用の予約がある場合は、重複登録されません。

## 4. 本日の予約を確認する

「本日の予約」タブを開き、当日の予約があるか確認します。

## 5. 今週の予約を確認する

「今週の予約」タブを開き、今週分の予約一覧を確認します。

## 6. LINE予約の動作確認で「予約」と入力する

「LINE予約の動作確認」タブを開き、送信メッセージで「予約」を選ぶか、任意メッセージに `予約` と入力します。

メニュー一覧が返ることを確認します。

## 7. LINE予約の動作確認で予約登録を試す

例：

```text
予約 30分相談 2026-05-15 10:00
```

この日付が過去日になっている場合は、現在日以降の営業日に置き換えてください。

## 8. 「確認」で直近予約を確認する

任意メッセージに次を入力します。

```text
確認
```

直近の予約情報が表示されることを確認します。

## 9. 「キャンセル」で直近予約をキャンセルする

任意メッセージに次を入力します。

```text
キャンセル
```

直近の予約がキャンセルされることを確認します。

## 10. 予約データをダウンロードする

「予約データをダウンロード」タブを開き、「予約データをダウンロード」を押します。

ファイル名は `reservations_YYYYMMDD.csv` 形式です。Excelで開きやすいようにUTF-8 BOM付きで出力されます。

## 11. 予約前のお知らせ予定を確認する

「予約前のお知らせ予定」タブを開き、現在から指定時間以内に通知対象となる予約を確認します。

LINE公式アカウントとつなぐ前でも、管理画面上で予約前のお知らせ対象の確認と処理の流れを確認できます。
